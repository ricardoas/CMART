Client

The C-MART client emulator access the website to create load on the servers. The emulated clients emulate modern browsing behaviors such as tabbed browsing, using AJAX, and upload and download images. The clients configuration is specified in CGconfig.txt.

Quick start

1. Configure the site IP address

full_url1=http://10.1.1.1:80/cmart-0.1
This variable tells the emulated clients which URL to use to open the C-MART application.

2. Configure the data directories

distributions_folder=C:\cmart\client\Distributions
The emulated clients use distributions taken from production websites to interact with the C-MART site. The location of the distribution files and the images used to send to the site and be found in the 'Distributions' folder of C-MART.

3. Configure the output directory

output_site_data_file=C:\cmart_output\
This variable specifies the directory used to output the response time, page hits and other statistics collected by the emulated clients throughout their sessions.

4. Configure the number of clients

clients_per_minute=50
peak_users=50
The clients_per_minute variable controls the ramp-up rate of the clients. This allows the web servers to 'warm up' before the peak number of users arrive. The peak_users variable controls the maximum number of emulated clients that access the site at any one time.

5. Configure the run length

run_length=60
The run_length variable is the number of minutes that the emulated clients will run before before they terminate, i.e. the experiment length.

6. Run the emulated clients

java -jar cmartclient.jar
As the number of images becomes very large it is likely the images will need to be stored on a dedicated or distributed storage system. Additionally, as multiple application servers will need to accesses the images they must be stored on shared storage.
