Populate Distributions

The C-MART populator is highly configurable and the data it creates can be edited via its configuration file or editing the data distribution files. This document describes how to configure the populator's distribution variables.

From file

1. File format

Frequency, Key, Value

Example:

25, 1, Eggs
50, 2, Bacon
25, 3, Sausage
When files are specified for distributions, such as populate_start_price_file, the database is populated using the distributions specified in the file. For example, in the above example, when a random value is chosen the key/value pairs (1, Eggs), (2, Bacon) and (3, Sausage) will be returned with the probability 25%, 50% and 25% respectively. The value column is optional if only the key is needed. For example, the starting price distribution could be defined as follows:

25, 5.00
25, 10.00
30, 15.00
10, 20.00
10, 25.00
The values in the first column will be normalized to 1 when values are drawn from the distribution so the raw frequency of values can be specified in the first column.

Specified distribution

2. Mathematical distribution

# -1= use file, 0 = uniform, 1 = normal
populate_bids_dist_type = 1
populate_bids_min = 0
populate_bids_max = 30
populate_bids_dist_mean = 10
populate_bids_dist_sd = 2
populate_questions_length_alpha = 1
populate_questions_length_lambda = 1
Rather than specifying a distribution in a file, a distribution can be specified using given parameters in cmartconfig.txt. Data is then created drawing from the mathematical distribution specified. Parameter values can be specified as double values, the final number chosen will be rounded to an integer value if/when required by the populator.
