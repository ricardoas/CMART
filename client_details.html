Client

The C-MART client emulator access the website to create load on the servers. The emulated clients emulate modern browsing behaviors such as tabbed browsing, using AJAX, and upload and download images. The clients configuration is specified in CGconfig.txt. This document describes how to configure the variables for the Client Generator.

Website Address and File Directories


1. full_url1, full_url2, full_url3, etc.

full_url1=http://10.1.1.1:80/cmart-0.1
This variable tells the emulated clients which URL to use to open the C-MART application. The format should be in the form of http://ip:port/appName. C-MART can send to multiple IP addresses to emulate DNS. full_url1 must always be set and any additional URLs, for example for multiple load balancers, should be set in order as full_url2, full_url3, etc.

2. distributions_folder

distributions_folder=C:\cmart\client\Distributions
The emulated clients use distributions taken from production websites to interact with the C-MART site. The location of the distribution files and the images used to send to the site contained in the 'Distributions' folder. These must be included for the Client Generator to run.

3. output_site_date_file

output_site_data_file=C:\cmart_output\
This variable specifies the directory used to output the response time, page hits and other statistics collected by the emulated clients throughout their sessions.

4. repeated_xml

repeated_xml=C:\cmart\repeatedXML\
This variable specifies the directory used to output the response time, page hits and other statistics collected by the emulated clients throughout their sessions.

5. output_stats

output_stats=true
The output_stats variable indicates whether statistics about the run should be output. Default is true.

6. output_matlab

output_matlab=true
The output_matlab variable indicates whether the data should be output in .mat form in addition to .csv form. Default is false.

7. output_think_times

output_think_times=true
The output_think_times variable indicates whether the distribution of think times is to be output. Default is false.

8. get_css_js_img

get_css_js_img=true
The get_css_js_img variable is used to indicate whether the CSS, JavaScript, and Image files on each webpage should be downloaded. If set to false only the HTML document for each page is downloaded. Default is true.

Runtime Variables


9. run_length

run_length=15
The run_length variable defines the length of the experiment in minutes. This includes rampup time at the start but does not include ramp-down time at the end when clients gracefully exit. Default is 10.

10. verbose

verbose=false
The verbose variable indicates if debugging output should be displayed. Options are 'true' or 'false'. Default is false.

11. html4

html4=true
The html4 variable indicates whether the HTML4 ('true') or HTML5 ('false') version of C-MART is to be used. Default is HTML4.

12. speedup_factor

speedup_factor=5
The speedup_factor is used to speedup the client's access to the website by changing the think time between page accesses. A speedup_factor of 5 would make the think time 1/5th the normal amount. A speedup_factor between 0 and 1 could be used to slow down the clients. Default speedup_factor=1.

13. repeated_run
repeated_run=false
The repeated_run variable is used to determine if the experiment run is based off repeating an existing run ('true') based off the XML files in repeated_xml, or if it is to be an original Client Generator experiment ('false'). If repeated_run is set to false, then the results of each client are saved to the repeated_xml directory for so that the run can be repeated. Default is false.

Client Load Variables


14. static_userload

static_userload=true
The static_userload variable is used to determine if the client generator is used in closed loop ('true') or open loop ('false') mode. In closed loop mode a client exiting the system spawns a new client to join, keeping the userload static. In the open loop mode clients are created at a specified rate. Default static_userload is false.

15. clients_per_minute

clients_per_minute=100
The clients_per_minute variable is used when static_userload is set to false. The value of clients_per_minute determines the average number of clients to be added to the system every minute. Default is 100.

16. peak_users

peak_users=300
The peak_users variable is used when static_userload is set to true. The value determines the value of the number of users to be held statically for the experiment. Default is 100.

17. rampup_time

rampup_time=60
The rampup_time is used for when static_userload is set to true. It is the amount of time in seconds over which the client level ramps up to the peak_users level. Default is 30.

18. allow_bursts

allow_bursts=false
The allow_bursts variable determines if C-MART is allowed to add in random client bursts to the workload. User defined bursts will be defined in subsequent sections.  Default is true.

19. change1, change2, change3, etc.

change1=30,600
change2=120,400
These variables are used to change the clients_per_minute variable over the course of the experiment for when static_userload is set to false. Each variable is a set of two numbers, the first number is the time in seconds from the start of the experiment when the change is set to begin, the second number is the new clients_per_minute value which should be implemented.

20. slope1, slope2, slope3, etc.

slope1= 30,30
slope2=60,-30
These variables are used to change the peak_users variable over the course of the experiment for when static_userload is set to true. Each variable is a set of two numbers, the first number is the time in seconds from the start of the experiment when the change is set to begin, the second number is the slope, in terms of clients/minute, of how much the peak_users value is to change.


Additional Variables


21. rt_threshold

rt_threshold=1000
The rt_threshold variable defines a threshold in milliseconds, above which the clients become dissatisfied with the response time and become more likely to leave the website. The rt_threshold is used as the average of a normal distribution for which each client has their own defined threshold value. Default is 1000.

22. tabbed_browsing

tabbed_browsing=true
The tabbed_browsing variable indicates if clients are allowed to open tabs in the style of modern Internet browsers.  Default is true.

23. markov_page_transitions

markov_page_transitions=false
The markov_page_transitions variable is used to determine if the page transition probabilities are determined by a Markov distribution on the page type, or if there is a more complex decision process involved to determine the page transition probabilities involving page content, client history, etc.  Default is false.

24. workload_type

workload_type=2
The workload_type variable indicates whether the workload type should be database read heavy ('1'), mixed ('2'), or write heavy ('3').  Default is 2.

25. cache_clear

cache_clear=0.33
The cache_clear variable determines what percentage of clients clear their local cache upon exiting the system. Clients returning to the system come with caches prepopulated from their previous browsing experience. Some clients will have empty caches due to possibly switching computers or specific browsing settings.  Default is 0.2.

26. cache_real_data

cache_real_data=false
The cache_real_data variable is used to determine if the real CSS, JavaScript, Image files are cached ('true') or whether there are just flags indicating that the specific files have been cached ('false'). Caching the real returned data is extremely memory intensive.  Default is false.

27. local_images

local_images=false
The local_images variable is used to indicate whether the images on C-MART are read from the local application server ('true') or from an external image server ('false').  Default is false.

28. network_delay

network_delay=false
The network_delay variable is used to add a fake addition network delay to each request. Since clients originate from the same source they all experience the same network delay and this can be used to differentiate between clients. Default is false.

29. network_delay_avg

network_delay_avg=50
The network_delay_avg variable is used when network_delay is set to true. The network_delay_avg is a value in milliseconds which is used as the average of an exponential distribution from which the network delay for each client is drawn from. Default is 0.
