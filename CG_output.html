Client

The C-MART client emulator access the website to create load on the servers. The emulated clients emulate modern browsing behaviors such as tabbed browsing, using AJAX, and upload and download images. The clients configuration is specified in CGconfig.txt. This document describes the files and statistics that are output during and after each run of the Client Generator.


ClientSessionStats

ClientSessionStats is a list of values describing each client's session. Each row in the table represents a different client. The column structure is as follows:

numPagesOpened
 RT avg
 requestErrors
 sessionLength
 exitDueToError
 dissatisfiedProb
  • numPagesOpened - The number of webpages the client opened during the session
  • RT avg - The average response time the client received from all page requests
  • requestErrors - The nnumber of errors the client had on page requests
  • sessionLength - The time length of the session in milliseconds
  • exitDueToError - Whether or not the client exited due to errors in the C-MART system
  • dissatisfiedProb - The probability that the client left C-MART due to disatisfaction with the service


numReqs

numReqs is a list of the number of active clients in the C-MART system and the number of requests they make during each 10-second interval. Each row in the table is a new 10-second time interval. The document is updated with a new row every 10 seconds. The column structure is as follows:

activeClients
 numRequests
  • activeClients - The number of active clients in C-MART at the end of the 10-second time interval
  • numRequests - The total number of requests made by all clients during the 10-second time interval


pageFreq

pageFreq is the number of each type of page request during each 10-second period. The document is updated every 10 seconds. The column structure is as follows:

pageRTHistograms

pageRTHistograms gives a histogram of the response times for each page type. Each row corresponds to a different page type and each column is the number of values in the histogram bin. The bin size is 2 milliseconds. The row structure corresponding to the page types is as follows:

thinkTimes

thinkTimes is a histogram of the think times of all the clients for the entire experiment run. The column structure is as follows:

thinkTimeBin
 numValues
  • thinkTimeBin - The bin value in milliseconds of the think times.
  • numValues - The number of instances of think times during the entire experiment that fell into the respective thinkTimeBin value.


responseTimes

responseTimes gives the time values for the response times for all clients for every 5th percentile in each 10-second interval. There are 20 columns in the table, corresponding to the 5, 10, 15, ... 90, 95, 100th percentile values for the response time in the time interval. Each row corresponds to a new 10-second time interval and the table is updated every 10 seconds.


siteData

siteData gives histograms on how long it takes to create each webpage. For each page type a histogram is given for how long it took to process the page parameters, access the database, process the database data, render the page, and the total for the entire process. The histograms are given by the columns where each column is a 10 millisecond interval. This data is only valid for when output_stats is set to false.
