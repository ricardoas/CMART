Populate

The C-MART data populator code can also perform some additional operations that may be useful such as compressing images, or populating the Solr server after the initial database has been populated.

Additional Operations

1. dbtosolr

java -jar cmartpopulator.jar -config c:\cmart/cmartconfig.txt -dbtosolr
Items that are currently for sale in the database are read and inserted into the Solr server. This is useful if the database was initially populated without the Solr server, but the Solr server is now required. The Solr server and database information are read from the config file. The number of items that are attempted to be inserted are the number of items specified in the configuration file. The number of old items must also be specified as it is used as the ID offset for the items table.

2. dbtoimg

java -jar cmartpopulator.jar -config c:\cmart/cmartconfig.txt -dbtoimg
If the database was populated without creating the items' images, the dbtoimg command can read each item in the database and create corresponding image files. dbtoimg uses parameters as creating images with the items, but it will read the items from the database and create the images, rather than creating a new item and then creating images.

3. compress

java -jar cmartpopulator.jar -config c:\cmart/cmartconfig.txt -compress
The compress command can be used to create compressed images and thumbnail images for the populator. Pre-compressed images are used to populate as it save having to compress each image when creating items - as happens when user's upload images during experiments. If you want to change the set of images C-MART uses you can use -compress to create the pre-compressed images. The parameters used by -compress are the same as used by the item image creation, i.e. the image in and out folders, and the list of images to be used.

4. noimg

java -jar cmartpopulator.jar -config c:\cmart/cmartconfig.txt -noimg
The noimg command allow the database to be populated without creating the corresponding item image files. This command can be used if disk I/O and image storage are not of interest in the experiments being run.
