Populate Details

The C-MART populator is highly configurable and the data it creates can be edited via its configuration file or editing the data distribution files. This document describes how to configure the populator's variables.

Users

1. populate_no_of_users

# create 100 users
populate_no_of_users =100

# create 1M users
populate_no_of_users = 1000000
This variable controls how many users will be created when populating C-MART's database.

2. populate_first_names_file

# DEFAULT: will use default first names list
populate_first_names_file = firstNames.csv

# will create use "custom_firstNames.csv" as input file
populate_first_names_file = custom_firstNames.csv
This variable controls which file is used to choose the first names of users.
The file format is: frequency with which to select the name, the name


3. populate_last_names_file

# DEFAULT: will use default last names list
populate_last_names_file = lastNames.csv

# will create use "custom_lastNames.csv" as input file
populate_last_names_file = custom_lastNames.csv
This variable controls which file is used to choose the last names of users.
The file format is: frequency with which to select the name, the name

4. populate_user_ratings_file

# DEFAULT: will use eBay emperical seller rating
populate_user_ratings_file = sellerRating.csv

# will create use "custom_lastNames.csv" as input file
populate_user_ratings_file = custom_sellerRating.csv
This variable control the distribution that is used for user's seller ratings.
The file format is: frequency with which to select the rating, the rating

User Addresses

5. populate_streets_file

# DEFAULT: will use popular American street names
populate_streets_file = streetNames.csv

# will create use "custom_streetNames.csv" as input file
populate_streets_file = custom_streetNames.csv
This variable controls the distribution used to selected the value of street names used in creating users' addresses.
The file format is: frequency with which to select the street name, the street name

6. populate_towns_file

# DEFAULT: will use popular American cities
populate_towns_file = cities.csv

# will create use "custom_cities.csv" as input file
populate_towns_file = custom_cities.csv
This variable controls the distribution used to selected the value of city names used in users' addresses.
The file format is: frequency with which to select the city name, the city name

7. populate_no_of_states

# DEFAULT: insert all states
populate_no_of_states = -1

# insert only 20 states
populate_no_of_states = 20
This variable controls the number of states inserted in to C-MART's database, typically all states should be added.

8. populate_states_uniform

# DEFAULT: will use emperical distribution of eBay buyers
populate_states_uniform = false

# will create peoples address with uniform states
populate_states_uniform = true
This variable controls if the user's state is selected from a uniform distribution or the one defined in states.csv

9. populate_states_file

# DEFAULT: use USA states
populate_states_file = states.csv

# will create peoples 'states' with custom values from "custom_places.csv"
populate_states_file = custom_places.csv
This variable sets the file to use for the user's states and the distribution that should be used to select user's locations.

The file format is: frequency with which to select the state, state short name, state full name

The order values appear in the file will be the same order they are inserted in to the database.

Items

10. populate_no_of_categories

# DEFAULT: will populate all eBay categories
populate_no_of_categories = -1

# will populate 100 eBay categories
populate_no_of_categories = 100
This variable controls how many item categories are inserted in to C-MART's database. The default is to insert all categories.

11. populate_categories_uniform

# DEFAULT: will select an item's category from the empirical distribution
populate_categories_uniform = false

# will select an item's category using a uniform distribution
populate_categories_uniform = true
This variable controls whether an item's category is selected using the observed eBay distribution or a uniform distribution

12. populate_categories_file

# DEFAULT: use eBay categories
populate_categories_file = categories.csv

# will create items with categories from "custom_categories.csv"
populate_categories_file = custom_categories.csv
This variable controls the file used to select items' categories and the distribution with which they are selected.
The file format is: frequency with which to select the category, category ID, category name, parent category ID

13. populate_start_price_file

# DEFAULT: use distribution from starting prices on eBay
populate_start_price_file = startingBid.csv

# will create items with categories from "custom_startingBid.csv"
populate_start_price_file = custom_startingBid.csv
This variable controls the file used to select item's starting prices.
The file format is: frequency with which to select the price, the price in dollars ($)

14. populate_buy_now_price_file = buyNowPrice.csv

# DEFAULT: use distribution from buy now prices on eBay
populate_buy_now_price_file = buyNowPrice.csv

# will create items with categories from "custom_buyNowPrice.csv"
populate_buy_now_price_file = custom_buyNowPrice.csv
This variable controls the file used to select item's buy now prices. The buy now price is always higher than the starting price and reserve price if the item is available to buy now.
The file format is: frequency with which to select the price, the price in dollars ($)

Database

15. Configure database credentials

# MySQL connection
my_database_url = jdbc:mysql://10.1.1.1/cmart
my_database_username = cmart
my_database_password = Cm4rt!
my_database_driver = com.mysql.jdbc.Driver

# Cassandra connection
cassandra_database_url = jdbc:cassandra://10.1.1.1:9160/CMART
cassandra_database_driver = org.apache.cassandra.cql.jdbc.CassandraDriver
cassandra_database_username = cmart
cassandra_database_password = Cm4rt!
The cmart user must be able to add/drop tables, create indexes, and insert, delete, update etc table entries.


16. Database connection cache

# Close each connection after query
my_conn_cache = 0

# Internal connection cache
my_conn_cache = 1

# Tomcat JDBC connection cache
my_conn_cache = 2
The cmart website can either not cache connections, keep track of its own connections or use a cache managed by the Tomcat context.
